## Tech Stack

| Package | Version | Role |
|---|---|---|
| `fastapi` | 0.135.1 | Web framework; auto-generates OpenAPI docs |
| `uvicorn` | 0.41.0 | ASGI server that runs the FastAPI app |
| `pydantic` | 2.12.5 | Request/response validation and serialisation |
| `supabase` | 2.28.0 | Python client for Supabase REST and Auth APIs |
| `python-dotenv` | 1.2.2 | Loads `SUPABASE_URL` and `SUPABASE_KEY` from `.env` |

## App Setup

The entry point is `server/main.py`. It creates the FastAPI application, registers CORS middleware, mounts the API routers, and sets up SPA serving:

```python
# server/main.py
app = FastAPI(title="Energy Contract Marketplace API")

app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

app.include_router(contracts.router, prefix="/api")
app.include_router(portfolio.router, prefix="/api")
```

<Note>
  `allow_origins=["*"]` is suitable for local development. Restrict this to your deployment domain in production.
</Note>

## Router Structure

The API is split into two routers, both mounted under `/api`:

### Contracts Router

| Method | Path | Description |
|---|---|---|
| `GET` | `/api/contracts` | List contracts with optional filters and sort |
| `GET` | `/api/contracts/trends` | Aggregated price trend data per energy type |
| `POST` | `/api/contracts/{id}/sell` | Mark a contract as sold |

### Portfolio Router

| Method | Path | Description |
|---|---|---|
| `GET` | `/api/portfolio` | Return current user's portfolio + metrics |
| `POST` | `/api/portfolio/{contract_id}` | Add a contract to the portfolio |
| `DELETE` | `/api/portfolio/{contract_id}` | Remove a contract from the portfolio |

## Pydantic Models

All request bodies and response shapes are defined in `server/models.py`:

```python
# server/models.py
class ContractBase(BaseModel):
    energy_type: str
    quantity_mwh: float
    price_per_mwh: float
    delivery_start: date
    delivery_end: date
    location: str
    status: str
    provider: Optional[str] = None
    description: Optional[str] = None
    carbon_intensity: Optional[float] = None

class EnergyContract(ContractBase):
    id: str                          # includes database-assigned ID

class ContractCreate(ContractBase):
    status: str = "Available"        # default status for new contracts

class PortfolioItem(BaseModel):
    id: str
    user_id: str
    contract_id: str
    contract: Optional[EnergyContract] = None

class PortfolioMetrics(BaseModel):
    total_contracts: int
    total_capacity_mwh: float
    total_cost: float
    avg_price_per_mwh: float
    breakdown: dict

class LoginRequest(BaseModel):
    email: str
    password: str
```

## Supabase Client

The Supabase client is created on demand via a factory function in `server/database.py`. Credentials are loaded from environment variables (populated by `python-dotenv` from `.env`):

```python
# server/database.py
import os
from dotenv import load_dotenv
from supabase import create_client, Client

load_dotenv()

def get_supabase() -> Client:
    url: str = os.environ.get("SUPABASE_URL", "")
    key: str = os.environ.get("SUPABASE_KEY", "")
    return create_client(url, key)
```

Routers import `get_supabase` and call it at the top of each handler. This keeps credential loading centralised and makes it straightforward to swap in a mock client for testing.

<Tip>
  Create a `server/.env` file with `SUPABASE_URL` and `SUPABASE_KEY` before starting the server. These values are available in your Supabase project dashboard under **Project Settings → API**.
</Tip>

## Serving the React SPA

After all API routes are registered, a catch-all route handles every remaining URL. This pattern lets the FastAPI process serve both the API and the compiled React app:

```python
client_dist_path = os.path.join(
    os.path.dirname(os.path.abspath(__file__)),
    "..", "client", "dist"
)

@app.get("/{path:path}")
async def serve_spa(path: str):
    file_path = os.path.join(client_dist_path, path)
    if path != "" and os.path.isfile(file_path):
        return FileResponse(file_path)      # serve .js / .css / assets
    index_path = os.path.join(client_dist_path, "index.html")
    if os.path.isfile(index_path):
        return FileResponse(index_path)     # SPA fallback
    return {"error": "Frontend build not found. Run 'npm run build' in the client directory."}
```

<Warning>
  API routes (`/api/*`) must be registered **before** the catch-all `/{path:path}` route. FastAPI matches routes in registration order, so any route added after the catch-all would be unreachable.
</Warning>

## Running the Server

```bash
# Install dependencies
pip install -r requirements.txt

# Start the server
python main.py
# or directly with uvicorn:
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

The server listens on `http://0.0.0.0:8000`. With the React build present in `client/dist/`, opening that URL in a browser loads the full application. The auto-generated OpenAPI docs are available at `/docs`.
